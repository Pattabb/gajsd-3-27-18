Hubot: Warm Up

We can interact with Hubot in a few ways:

• Listening:

.hear

.respond

• Speaking:

.send

.reply

• Interactions leverage Regular Expressions (RegEx)

Hubot: Hear

.hear

• Listens for a specific phrase anywhere in the Slack room

• You don't have to mention your Hubot in order to get a response.


	robot.hear(/hello|hi/i, function(res) {
	  return res.send("Hello, World!");
 	});

 	// user: Hello
 	// hubot: Hello, World!
Hubot: Respond

.respond

• Similar to hear, but Hubot only responds to direct messages and @-mentions


  robot.respond(/What's your favorite food?/i, function(res) {
    return res.send("I'm a robot; I don't eat food!");
  });

  // user: @hubot What's your favorite food?
  // hubot: I'm a robot; I don't eat food!
  // user: What's your favorite food?
  // ... [no response because the user didn't tag Hubot in the message]
Hubot: Send and Reply

.send

• Hubot will send a message to the entire channel


  robot.respond(/What's your favorite food?/i, function(res) {
    return res.send("I'm a robot; I don't eat food!");
  });

.reply

• Hubot will reply with an @-mention

• Requires one (or more) wildcard selector(s)

Hubot: Replying

• Use a wildcard selector - (.*) - to accept user-input

• Input will be storred in an array, match, which lives in res


  robot.respond(/My name is (.*)/i, function(res) {
	var name = res.match[1];

	if (name === "Hubot"){
 	  return res.send("You're not Hubot--I'm Hubot!");
	} else {
 	  return res.reply("Nice to meet you, " + name + "!");
	}
  });
Hubot: Replying

• To capture additional input, use more wildcard selectors:


	robot.respond(/add (.*) and (.*)/i, function(msg) {
	  var x = parseInt(msg.match[1]);
	  var y = parseInt(msg.match[2]);
	  var sum = x + y;

	  return msg.reply(x + " plus " + y + " = " + sum);
	});
