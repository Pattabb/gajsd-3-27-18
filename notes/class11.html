Geolocation

Geolocation: Navigator

• Modern browsers have a navigator object

• Used to access/identify users' browser information

• Available in console via navigator:

navigator.language ⇒ user's language ex: en-US

navigator.platform ⇒ user's platform ex: MacIntel

navigator.language ⇒ users's browser ex: Google Inc.

Geolocation

• navigator also contains a geolocation object

• Enables users to share geolocation data with web applications

• Users must enable the feature on a per-site basis

• Note: not all browsers have this feature:


  if (navigator.geolocation) {
    /* user has geolocation */
  } else {
    /* user does not have geolocation */
  }
Geolocation

• getCurrentPosition returns users' positions

• Note: this is an asynchronous request; requires a callback

• The callback needs an arugment for the position


  navigator.geolocation.getCurrentPosition(handleResponse)

  // callback
  function handleResponse(position) {
    console.log(position)
  }

  // {
  //  coords: {
  //    latitude: 40.6,
  //    longitude: -73.9,
  //    ...
  //  },
  //  timestamp: 1521422644286
  // }

Function Creation

Function Creation

• JS provides multiple ways to create functions

• Named: functions associated with a name that can be referenced later

• Anonymous: functions with no name; exist only when running


  // named function
  function hello() {
    conosle.log('hello, world!');
  }

  // anonymous function
  function() {
    console.log('hello, world!');
  }
Named Functions

• Named functions can be created in two ways:

⇒ Expressions

⇒ Declarations


  // expression
  var sayHello = function(greeting) {
    console.log(greeting)
  }

  // declaration
  function sayHello(greeting) {
    console.log(greeting)
  }

  // both of the above can be called as:
  sayHello('hello, world!')
Function Expressions

• Function expressions are hoisted

• Hoisted functions are loaded in memory during compilation

• We can use hoisted functions "before" we declare them:


  // invoked before declaration
  sayHello('hello, world!')

  function sayHello(greeting) {
    console.log(greeting)
  }
Anonymous Functions

• Anonymous functions are functions without a name

• Created during application runtime

• Convenience feature allowing easy function creation

• Useful for callbacks

Anonymous Functions: Callback Examples

• Handling a browser event:


  $('#my-btn').click(function() {
    // do something
  })

  $(document).on('click', 'li', function() {
    // do something
  })

• Handling an asynchronous event:


  $.ajax({
    type: 'GET',
    url: https://my-api.com,
    success: function(res) {
      console.log(res)
    }
  })
Anonymous vs Named Functions

• Anonymous functions are a convenience feature

• Named functions could be considered better practice:

⇒ Useful in recursive situations

⇒ Will appear named in callstack during debugging

⇒ Forces developer to consider what the funciton does

First-Class Functions

Functions as First-Class Objects

• A programming language supports first-class functions if:

⇒ Stored in a variable

⇒ Passed as a parameter to another function (callback!)

⇒ Returned from a function

⇒ Included in data structures (objects and arrays)

• AKA - the language treats functions just like any other data type

Functions as First-Class Objects

• Higher-order functions:

⇒ Functions that take other functions as an argument, or return functions

• One of JS's super powers (see: functional programming)

• JS has a few higher-order functions baked in:


  setTimeout(function() {
    console.log("Hello, World!");
  }, 1000);

• setTimeout() takes two arguments:

⇒ A function to be called/invoked

⇒ The duration to wait before invoking the function

Functions as First-Class Objects

• No difference in syntax for higher-order functions:

• Create a https://repl.it/ with:


  var blastOff = function() {
    console.log('Blasting off!')
  }

  function launchRocket(rocketName, blastOffCallback) {
    console.log('Launching ' + rocketName)
    console.log('3... 2... 1...')
    blastOffCallback()
  }

  launchRocket('Viking', blastOff)

  // `Launching Viking
  // 3... 2... 1...
  // Blasting off!
Functions as First-Class Objects

• In the previous example, blastOffCallback is a callback

• Callbacks are functions that are called by another function

• Callbacks can take arguments:


  var blastOff = function(destination) {
    console.log('Blasting off for ' + destination + '!')
  }

  function launchRocket(rocketName, blastOffCallback) {
    console.log('Launching ' + rocketName)
    console.log('3... 2... 1...')
    blastOffCallback('Mars')
  }

  launchRocket('Viking', blastOff)

  // Launching Viking
  // 3... 2... 1...
  // Blasting off for Mars!
Functions as First-Class Objects

Functions can also be returned from other functions:


  function makeRocketLauncher(rocketName) {
    return function(destination) {
      console.log('Launching ' + rocketName)
      console.log('3... 2... 1...')
      console.log9('Blasting off for ' + destination + '!')
    };
  }

  var launchViking = makeRocketLauncher('Viking')
  var launchMariner = makeRocketLauncher('Mariner')

  launchViking('Mars')
  launchMariner('Mars')

  // Launching Viking
  // 3... 2... 1...
  // Blasting off for Mars!

  // Launching Mariner
  // 3... 2... 1...
  // Blasting off for Mars!
